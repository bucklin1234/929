Key Concepts

    
Click any card to expand

    

      

        

          🧩
          What is a Token?
          ›
        

        

          
A token is the smallest unit a model reads and writes. It can be a word, part of a word, or punctuation.


          "Hello, world!" → 4 tokens

          "unbelievable" → 3 tokens (un + believ + able)


          On average, 1 token ≈ 0.75 English words. Models have a context window — a maximum total tokens they can process in one request. You pay per token used.

        

      


      

        

          🌡
          What is Temperature?
          ›
        

        

          
Temperature controls randomness. At each step the model has a probability distribution over all possible next tokens.


          temperature: 0 — always picks the highest probability token. Deterministic, consistent.

          temperature: 1 — samples from the distribution naturally.

          temperature: 2 — flattens the distribution, more surprising/creative output.


          For factual tasks use 0.1–0.3. For creative writing try 0.8–1.2.

        

      


      

        

          💬
          What is the Messages Array?
          ›
        

        

          
LLMs are stateless — they have no memory between requests. To maintain a conversation, you send the entire history with every request.


          Each message has a role:

          "system" — instructions that shape the model's behaviour

          "user" — the human's message

          "assistant" — the model's previous reply


          You keep appending to this array as the conversation grows.

        

      


      

        

          ⚡
          Streaming vs Batch
          ›
        

        

          
By default (stream: false), the API waits until the entire response is generated, then sends it all at once as one JSON object.


          With stream: true, the API sends each token as a Server-Sent Event (SSE) the moment it is generated — creating the familiar "typing" effect in chat interfaces.


          Streaming feels faster to the user even though total generation time is the same. It also lets you handle very long responses without timeout issues.

        

      


      

        

          🛑
          stop_reason Explained
          ›
        

        

          
The response includes a stop_reason field telling you why generation stopped:


          "end_turn" — the model naturally finished its answer

          "max_tokens" — hit your max_tokens limit mid-sentence

          "stop_sequence" — matched a custom stop string you provided


          Always check this field. If it says max_tokens, the response is truncated and you may need a higher limit.

        

      


      

        

          🔐
          API Keys & Auth
          ›
        

        

          
Every request must include your secret key in the x-api-key header. This identifies you and your billing account.


          Never put your API key in frontend JavaScript — it will be visible to anyone who views page source. Always call the API from a backend server that keeps the key secret.


          If a key is leaked, rotate it immediately in the Anthropic console.

01

Concept Overview

A multi-cuisine food cafe operating 9 AM – 9 PM daily ("9 to 9"), with 9 cuisine stations arranged in a semicircular layout. Customers browse each station, add items to a shared cart, and pay at a single central checkout point.

The web application mirrors this physical store experience and serves as the launch vehicle before the physical store opens. The UI directly reflects the intended floor plan — a semicircular arc with radiating segments, a central counter, and a checkout anchored to the right.

Operating Rules

Operating Hours
9:00 AM – 9:00 PM daily
Outside Hours
Store shows "Closed" state — browsing allowed, ordering disabled
Menu Cadence
Changes overnight via predefined rotation cycle
Ordering Model
Customer collects from individual stations, pays at one checkout
Consumption Model
Takeout only — no dine-in

Single central checkout — all cuisines combined into one transaction

Web Application — Feature Requirements

4.1  Pages

Home / Store View

Semicircular station map — the primary browsing and ordering experience

Checkout

Cart review and order placement with grand total

Contacts

Static info: phone, email, address, operating hours

4.2  Home / Store View

Layout

  • Render a semicircular arc with 9 equal radiating segments
  • Each segment is labelled with its cuisine station name
  • A central counter element sits inside the arc (visual landmark)
  • Entrance indicator at the flat bottom edge of the arc
  • Checkout button anchored at bottom-right

Segment Interaction

  • User clicks a segment → that segment expands / pops out to reveal its item list
  • All other 8 segments gray out — visually dimmed and non-interactive while one is open
  • User can close the expanded segment to return to full arc view
  • Only one segment can be expanded at a time

Item Selection (within an expanded segment)

  • Today's menu items listed: name, description, price, optional image
  • Each item has an Add to Cart button
  • Multiple items from the same station can be added before closing
  • Quantity adjustable with + / − controls

Cart Behaviour

  • Cart persists across station browsing — does not reset when a segment closes
  • Floating cart indicator (item count + running total) always visible
  • Cart panel accessible at any point without navigating away
  • Cart displays: item name, station, quantity, line total
  • Cart displays: grand total
  • Proceed to Checkout action in cart panel

4.3  Checkout View

  • Full summary of all cart items grouped by station
  • Grand total displayed prominently
  • Order confirmation action (payment integration deferred to a later phase)

4.4  Contacts Page

  • Static content: store name, address, phone, email
  • Operating hours: 9 AM – 9 PM
  • No contact form (deferred)

4.5  Operating Hours Gate

  • If current time is outside 9 AM – 9 PM, the store shows a "We're Closed" state
  • Ordering is disabled — Add to Cart buttons hidden or disabled
  • Menu browsing remains available in read-only mode

06

Prototype Scope

In Scope

  • Semicircular 9-station UI
  • Segment pop-out & gray-out interaction
  • Today's menu items per station (seeded data)
  • Add to cart, quantity management
  • Running total & cart panel
  • Checkout summary view
  • Operating hours gate (open/closed state)
  • Contacts page (static)

Out of Scope (Prototype)

  • Live payment processing
  • Admin panel & menu management
  • User authentication
  • Ready time / queue display
  • Live menu rotation (use seeded data)
  • Contact form
  • Mobile-native app
